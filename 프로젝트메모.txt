** github **

  팀장, 팀원 모두
    1. 팀장 : github -> porjects
    pull request 된 내용 코드 리뷰
    * 코드리뷰
      - 요구 사항 구현 확인
      - 불필요, 잘못된 것 체크
        tip : 

  브랜치(branch) : 원본 복제
    - 지정된 브랜치를 복제하여 새로운 브랜치를 생성
    - 원본을 유지하기 위함
    - 나중에는 달라진 부분만 병합 가능함 
    - 만들어진 브랜치의 변경된 내용을 원본 branch에 병합(merge)할 수 있음
    * 브랜치를 만들기 위해선 이슈 등록 먼저 해야함 *
    - 브랜치 이름 형식 -> feature-A테스트
    - 기능별 브랜치 만들었으면 소스트리에서 패치
    - 로컬 브랜치 생성(각자 이름으로)
    - 좌측 사이드 메뉴에서 만든 브랜치 우클릭, 원격 브랜치 추적
    - 원격 브랜치 선택

  브랜치 작업
    - 내용 작성 후 푸쉬(하기전에 400번 생각하고 하기)
    - 깃에서 pullRequest 작성 : 구현된 기능에 대한 요약 정리
    - 문제 없으면 팀장이 merge pull request 실행
    - (팀장)완료된 원격 브랜치 제거(삭제)
      - main -> 새 브랜치(A) -> main에 병합(main + A) -> 새 브랜치A는 main 병합 완료 후 제거
